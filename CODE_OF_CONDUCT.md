# Code of Conduct

YggdraSIM is a small, technical, and standards-driven project. The
maintainers expect every contributor, reviewer, issue author, and
discussion participant to follow the rules below.

## Our pledge

We pledge to make participation in YggdraSIM a harassment-free
experience for everyone, regardless of age, body size, disability,
ethnicity, gender identity and expression, level of experience,
nationality, personal appearance, race, religion, or sexual identity
and orientation.

## Expected behaviour

- Stay technical. Cite specifications (`GSMA SGP.02 / 22 / 32`,
  `GlobalPlatform Card Specification 2.3`, `ETSI TS 102.221 / 222 /
  223 / 226`, `3GPP TS 31.102 / 31.115 / 31.116`, `ISO/IEC 7816`)
  rather than personal preference.
- Be precise about scope. APDU / TLV evidence, file paths, line
  numbers, and reproducible steps move conversations forward faster
  than opinion.
- Respect the spec. If a behaviour deviates from the standard, surface
  the deviation (`[KNOWN SPEC DELTA]`) instead of working around it
  silently.
- Treat newcomers gently. Smart-card stacks have a steep onboarding
  curve; help people level up rather than dismissing them.
- Disclose conflicts of interest. If a contribution touches a vendor
  or operator you are affiliated with, say so in the issue / PR.

## Unacceptable behaviour

- Harassment, threats, or personal attacks of any kind.
- Public or private sexualised language or imagery.
- Sustained disruption of issue threads, pull requests, or chat.
- Publishing other contributors' private information without explicit
  permission.
- Submitting operator secrets, real subscriber identities (`IMSI`,
  `ICCID`, `EID`), production `AES` / `DES` / `ECC` keys, certified
  profile bundles, or vendor-confidential APDU traces -- see
  `CONTRIBUTING.md` for the data-hygiene rules.
- Using the project to attack, surveil, or compromise live mobile
  network infrastructure or third-party subscribers. YggdraSIM is a
  research and lab toolkit, not an offensive operations platform.

## Scope

This Code of Conduct applies to:

- the GitHub repository (issues, pull requests, discussions, code
  review, commit messages);
- any communication channel where a maintainer represents the project
  (mailing list, conference Q&A, official social media);
- offline contexts (events, workshops) where YggdraSIM is presented
  on behalf of 1oT OÜ.

## Enforcement

Reports of unacceptable behaviour can be sent to
**`conduct@1ot.com`** with the subject prefix
`[YggdraSIM][CONDUCT]`. For security-relevant reports, use the
process in `SECURITY.md` instead.

The maintainer will:

1. Acknowledge receipt within **5 working days**.
2. Investigate while protecting reporter anonymity to the extent
   practical.
3. Respond with one of: no action (with reasoning), private warning,
   public warning, temporary ban from the repository / channel, or
   permanent ban.

Decisions are final, but a reporter or accused contributor may request
a single review by sending a follow-up email within **14 days** of the
decision.
