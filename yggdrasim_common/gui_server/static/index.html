Shell

Spawn a registered CLI module inside a server-side PTY and drive it from the browser via xterm.js. Only modules listed in yggdrasim_common.registry.CLI_MODULES are accepted — the GUI cannot invoke arbitrary commands.

idle

Paste works; Ctrl+C sends SIGINT to the child; resizing the browser window re-fits the grid automatically. Opening another module spawns it in a new tab, so several PTYs can run side-by-side.

Host shell

Free-form interactive shell on the GUI host — equivalent to an SSH session as the user that launched yggdrasim. Use for ad-hoc modem CLI workflows (tio / minicom / socat against /dev/ttyUSB*). Disabled by default; flip YGGDRASIM_GUI_HOST_SHELL=1 to enable.

Host shell is disabled.

Set YGGDRASIM_GUI_HOST_SHELL=1 in the launcher env and restart yggdrasim-gui / yggdrasim-web-server.

idle
AT decode

Free-form RCE-class surface — the bearer token grants shell-equivalent access. Restrict who holds it and prefer an SSH tunnel over a public bind.

About this GUI

YggdraSIM Universal GUI — Phase A scaffolding for roadmap entry R2-004.

  • Built on FastAPI + a handwritten no-build SPA.
  • Token-gated; same-origin only; CSP locked to self.
  • Engine code stays headless — every endpoint resolves via yggdrasim_common.registry.
